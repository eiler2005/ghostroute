# Telegram: подробный разбор

## Почему Telegram — особый случай

Большинство сервисов (YouTube, GitHub, ChatGPT) работают через DNS: клиент запрашивает домен, получает IP, подключается. Для таких сервисов достаточно добавить домен в dnsmasq — и всё.

Telegram устроен иначе. Помимо DNS-резолвимых доменов, клиенты Telegram:

- подключаются напрямую к IP-адресам из собственных подсетей (AS62041, AS62014)
- получают список IP дата-центров при первом подключении и кэшируют его
- используют MTProto-протокол, который DPI может распознавать и замедлять

Поэтому для Telegram используется **три уровня маршрутизации**:

1. **Домены** — покрывают web-интерфейс, CDN, API, экосистемные сервисы
2. **Официальные подсети** — IP-диапазоны дата-центров Telegram
3. **AS62041 peering / CDN** — подсети, используемые для медиа-доставки в России

## Проблема DPI-троттлинга в России

РКН не просто блокирует Telegram по DNS или IP. Используется DPI (Deep Packet Inspection), который распознаёт Telegram-трафик на уровне протокола и замедляет его. Типичные симптомы:

- Текстовые сообщения отправляются нормально
- Фото и видео грузятся десятками минут или не грузятся вовсе
- Проблема непостоянна — зависит от провайдера, региона и времени суток

Подробное обсуждение проблемы: [Flowseal/zapret-discord-youtube#5820](https://github.com/Flowseal/zapret-discord-youtube/issues/5820)

| Домен | Назначение |
|---|---|
| `telegram.org` | Основной сайт, web-клиент, API (`web.telegram.org`, `api.telegram.org`) |
| `telegram.com` | Альтернативный домен (редирект на telegram.org) |
| `t.me` | Короткие ссылки |

| `telegra.ph` | Telegraph — платформа статей |
| `telesco.pe` | Web-представление публичных медиа |
| `tg.dev` | Технические и служебные хосты |

| Домен | Назначение |
|---|---|

| `telegram-cdn.org` | Дополнительное CDN-семейство для прямых файловых ссылок |

### AS62041 peering / CDN

Эти подсети не входят в официальный `cidr.txt`, но анонсируются через автономную систему Telegram (AS62041) и используются как CDN/пиринг для доставки медиа:

```
5.28.192.0/21
5.28.248.0/21
95.161.64.0/19
```

**Про 95.161.64.0/19:**

Формально диапазон принадлежит компании Global Network Management (GlobalNet), но фактически анонсируется через AS62041 (Telegram). В официальном `cidr.txt` отсутствует.

Зачем он нужен: сеть используется Telegram как CDN и точки пиринга для ускорения доставки контента (фото, видео) в российском регионе. Без неё Telegram будет работать, но медиафайлы могут грузиться значительно медленнее — или не грузиться вовсе при жёстких блокировках.

Wireshark-анализ живого трафика показывает `/19` (шире, чем рекомендуемый `/20`). Используем `/19` для полноты покрытия.

**Про 5.28.192.0/21 и 5.28.248.0/21:**

Подсети зафиксированы при Wireshark-анализе Telegram-сессий. Относятся к инфраструктуре доставки контента.

## Почему shared CDN IP не роутим статически

При Wireshark-анализе видно, что часть web-трафика Telegram проходит через shared CDN-провайдеров (Cloudflare, Google, Fastly). Эти IP-адреса задокументированы в `configs/static-networks.txt` как справочная информация, но **не добавлены как статические маршруты**.

Причина: это общая CDN-инфраструктура. Добавление, например, `172.67.132.10/32` в статические маршруты отправит через VPN весь трафик к этому IP — включая другие сервисы, не связанные с Telegram. Telegram web-трафик к CDN уже покрыт доменным роутингом (`telegram.org`, `t.me` и т.д.).

## IPv6

IPv6-подсети Telegram задокументированы в `static-networks.txt` как закомментированные записи:

```
# 2001:67c:4e8::/48
# 2001:b28:f23c::/46
# 2a0a:f280::/29
```

Для активации IPv6-роутинга нужно:

1. Убедиться, что WGC1 несёт IPv6-трафик (`ip -6 route show table wgc1` показывает маршруты)
2. Раскомментировать закомментированный блок IPv6 в `scripts/firewall-start`
3. Раскомментировать блок IPv6 в `scripts/nat-start`
4. Раскомментировать IPv6-подсети в `configs/static-networks.txt`
5. Задеплоить

## Как обновлять подсети

1. Скачайте актуальный список: `curl -s https://core.telegram.org/resources/cidr.txt`
2. Сравните с секцией «Официальные» в `configs/static-networks.txt`
3. Если есть расхождения — обновите файл и задеплойте

Для BGP-анализа можно использовать:
- https://bgp.he.net/AS62041 — маршруты AS62041 (Telegram)
- https://bgp.he.net/AS62014 — маршруты AS62014 (Telegram)
